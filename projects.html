Project template

The format stays simple.

If the page cannot explain the project in four blocks, the project is probably still a fog machine with bullet points.

01 / Why

State the itch.

What problem, aesthetic, or future-interface suspicion made the build necessary?

02 / Process

Show the mechanism.

Tools, agents, prompts, assets, constraints, and the parts that behaved badly.

03 / Learned

Extract the pattern.

What got sharper after the artifact existed? What would we steal next time?

04 / Proof

Attach receipts.

Screenshot, demo link, trace, commit, video, or artifact. The work becomes the proof.
